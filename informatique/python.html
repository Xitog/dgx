1. Introduction

Caractéristiques générales

Date d'apparition du langage : 1990

Créateur : Guido van Rossum (Pays-Bas).

Ceci est un brouillon d’une éventuelle feuille de notes pour Python.

  • Typage Fort + Dynamique (duck/latent typing)
  • Interprété, Cross-platform
  • Implémentations : CPython (C), Jython (JVM), IronPython (.Net), PyPy (Python)
  • Sous-langages :
    • Stackless Python (sans pile C)
    • RPython (sous ensemble de Python compilé pour PyPy)
    • Pyrex[->Cython] (type C + Python)
    • Extensions : .py, .pyc (bytecode), .pyx (Pyrex)

Affichage

Une fonction à connaître en Python est print. Elle permet d'afficher sur la sortie standard. Ainsi, l'application standard lorsqu'on apprend un nouveau langage de programmation s'écrira :

print("Hello, World!")

2. Types de base

1. Scalaires

A) Entiers

Ils peuvent être définis selon trois formats : hexadécimal (en base 16) en précédant le nombre avec 0x ou 0X, octal (en base 8 ) en précédant le nombre d’un 0 et en base 10. Il faut donc faire bien attention à repérer si le littéral commence par un 0 ou pas pour ne pas confondre base octale et décimale. Les chiffres supplémentaires en base 16 sont représentés par les lettres A à F (indifféremment de la casse).

Exemple : 8 (base 10), 010 (8 en base octale), 0xA (10 en base hexadécimale).

Un entier sera convertit en long si besoin est de façon automatique. Le long permet de ne pas avoir de limite relative à la grandeur du nombre représenté – celle-ci est en effet illimité. Le suffixe « L » ou « l » permet de déclarer d’office un long, même si Python se charge complètement et d’une manière transparente de la conversion.

Exemple : 8L

Règles de grammaire associées :

digit ::=
             "0"..."9"

longinteger ::=
             integer ("l" | "L")

integer ::=
             decimalinteger | octinteger | hexinteger

decimalinteger ::=
             nonzerodigit digit* | "0"

octinteger ::=
             "0" octdigit+

hexinteger ::=
             "0" ("x" | "X") hexdigit+

nonzerodigit ::=
             "1"..."9"

octdigit ::=
             "0"..."7"

hexdigit ::=
             digit | "a"..."f" | "A"..."F"

Pour contrôler sur combien de position s'affiche un entier décimal, on peut utiliser les puissantes possibilités de formatage de Python. Pour afficher ainsi le chiffre 5 sur 2 positions, en mettant 0 dans les positions inoccupées par le chiffre, il suffira d'écrire :

print(f"{5:02d}")

Le formatage commence par : puis le 0 indique le caractère à mettre dans les positions inoccupées, le 2 indique le nombre de positions, et enfin d qu'il s'agit d'un affichage décimal.

B) Nombres réels

Les nombres réels n’utilisent que la base décimale, est le « . » pour séparer la partie entière de la décimale. Le « . » est nécessaire, même sans partie décimale, pour différencier un entier ou un flottant. De même la partie entière, si elle est égale à 0, peut être omise. Un exposant peu être spécifié avec la lettre E (indifféremment de la casse).

Exemple : 3.4, 3., .3, 3.e20

Règles de grammaire associées :

floatnumber ::=
             pointfloat | exponentfloat

pointfloat ::=
             [intpart] fraction | intpart "."

exponentfloat ::=
             (intpart | pointfloat)
              exponent

intpart ::=
             digit+

fraction ::=
             "." digit+

exponent ::=
             ("e" | "E") ["+" | "-"] digit+

C) Nombres complexes

Il suffit d’utiliser le suffixe J (casse indifférente) pour indiquer une partie imaginaire. L’addition d’une partie réelle et imaginaire permet d’exprimer un nombre complexe.

Exemples : 3+2j, 3-2j

Règles de grammaire associée :

imagnumber ::= (floatnumber | intpart) ("j" | "J")

D) Booléens

Il n’y a que deux valeurs.

Exemples : True, False

E) Règles de grammaire associées aux littéraux

atom ::=
             identifier | literal | enclosure

literal ::=
             stringliteral | integer | longinteger
                | floatnumber | imagnumber

F) Énumérations

Il est possible de déclarer des énumérations en Python. Une énumération est un type dont l'ensemble des valeurs possibles est connu.

from enum import Enum
class MyEnum(Enum):
    V1 = 1
    V2 = 2

L'affichage d'une énumération n'est pas simplement sa valeur.

Une valeur d'énumération est hashable donc utilisable comme clé dans un dictionnaire.

On peut itérer sur les différentes valeurs d'une itération avec un for.

Séquences Immutables

  • str ’Hello’ | « Hello » immutable séquence de caractère
  • unicode u’Hello’ | u »Hello » immutable séquence de caractère
  • tuple (4.0, ‘a’, ‘b’) | (3,) immutable, on peut mixé les types
  • frozenset frozenset([4.0, 'a', True]) immutable, sans ordre, sans doublon

Séquences Mutables

  • list [1, 'a', True] mutable, on peut mixer les types
  • dict {‘key1' : 1.0, ‘key2' : False} paire clé / valeur
  • set set([4.0, 'a', True]) mutable, sans ordre, sans doublon

On peut faire des properties en Python, à la manière de Ruby :

class Pipo(object):
	...
    myProp = property(getter, setter, deleter, doc)

où chacun des éléments est une fonction. ActiveState a pondu un truc infâme pour éviter d’encombrer l’espace de nom de la classe des fonctions de getter/setter/etc. :

    def Property(func)
      return property(**func())
    class Person(object):
      @Property
      def name():
          doc = "The person's name"
          def fget(self):
              return "%s %s" % (self.first_name, self.last_name)
          def fset(self, name):
              self.first_name, self.last_name = name.split()
          def fdel(self):
              del self.first_name
              del self.last_name
          return locals()

VII. Gestion des exceptions

Bloc try / except / else / finally

try:
    actions
except TypeException1 as Identifiant:
    actions
except TypeException2:
    actions
else:
    actions si pas d'exception
finally:
    actions toujours exécutées

On peut attraper un type d'exception et mettre l'exception dans une variable avec as.

Le code défini dans else n'est exécuté que s'il n'y a pas eu d'exceptions et qu'il n'y a pas d'instructions comme break, continue ou return dans le code du try. Cela permet de garder le code gérant l'exception dans le bloc except proche de la ligne pouvant lever l'exception.

Le code défini dans finally est toujours exécuté, qu'il y ait eu une exception ou non.
